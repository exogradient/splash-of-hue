## Decisions
- **Homepage is a redesign, not a polish pass**: the vertical mode list didn't communicate the game's identity. 2-column card grid with a brand-mark hero is a different information architecture — treat as WIP, not ship.
- **PWA metadata ships before custom mode icons**: manifest.json and favicons are table stakes for mobile add-to-homescreen. Per-mode SVG icons are beta polish.
- **iPhone Safari needs its own viewport contract**: `viewport-fit=cover` + safe-area padding + top-aligned flex layouts. "Works on Pixel" is not "works on mobile."
- **Focus-visible over focus**: only keyboard/assistive users see the ring — mouse clicks stay clean.
- **Spectral ring as brand element**: the hue wheel says "color perception" instantly, while surface-specific treatments keep installed icons and tab favicons legible in their actual contexts.

## Vision
- **The homepage is the first teaching moment.** Before players pick a mode, the layout and brand mark should communicate that this is about perception, not trivia.
- **PWA is the delivery mechanism for a training tool.** Add-to-homescreen makes daily practice frictionless — the game should feel native, not like a website.

## Direction

**Durable:**
- Mobile-first means iPhone Safari first — it has the strictest viewport behavior
