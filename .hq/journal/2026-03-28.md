## Decisions

### Doc system & architecture
- **Four-layer doc system**: identity / design / specs / meta. Layer encoded as filename prefix. Three principles: non-duplicative, colocated, can't go stale.
- **PIT docs**: Frozen research snapshots in `docs/pit/`, not managed docs. Feed into layer system but not part of it.
- **Specs pipeline**: roadmap (prioritized direction) → features (committed specs) → journey (shipped flow).
- **CIEDE2000 scoring**: Sigmoid curve `10 / (1 + (ΔE00/20)^2.5)`, per-dimension deltas (ΔL', ΔC', ΔH'). More perceptually accurate than dialed's CIE76.
- **Results: clean by default**: Swatches + scores visible. HSB, ΔE, feedback text behind hidden activation (URL param / shortcut).
- **Feedback messages server-side**: Returned with submit response. Enables sharing/leaderboards showing the roast text.

### Visual identity
- **Achromatic constraint**: UI must be chromatic-free. Current navy/coral violates this. Move to near-black `#0a0a0a` with neutral gray ramp. Game colors are the only chromatic element.
- **Type is the brand**: Single distinctive typeface, single weight (500). Warm geometric or humanist sans — not Swiss brutalism (dialed's territory). Candidates: Satoshi, General Sans, Space Grotesk, Instrument Sans.
- **Evolve own identity, not clone dialed.gg**: Use dialed as quality gate — if it doesn't hold up side-by-side, not ready. But different direction: teaching, not intimidating.
- **SB field picker kept**: Spatial advantage over dialed's three strips. Polish, don't replace.
- **Desktop goes full-width**: Not card-in-viewport. Educational modes need breathing room.

### Skill model
- **Picker fluency is UX, not a skill**: Good UI makes it invisible. Skills are cognitive.
- **Recall → Retention**: "Recall" names only retrieval. Maintenance during delay is the bottleneck.
- **Context added as fifth core skill**: Simultaneous contrast awareness. Currently a gap — no mode targets this.
- **Scientific basis column**: Identity doc uses precise terminology (chromatic discrimination, color appearance scaling, color visualization, delayed reproduction, simultaneous contrast awareness).
- **Dial It In → Picture It**: Renamed. Multiple choice (pick swatch from HSB values), not picker-based. Pure decoding. "Picture It" names the cognitive operation and fits the verb-it naming pattern.

## Insights
- **Encoding is genuinely hard**: Untrained observers perceive color holistically. Dimensional decomposition into H/S/B is a learned skill — strong framing for launch post.
- **Transfer is hue-specific**: Discrimination training in one color region doesn't help others. Game should systematically cover color space.
- **Categorical drift during delay**: Color memories pull toward prototypes ("blue", "green") during retention. HSB encoding directly counteracts this — the pedagogical core of the app.
- **Sapir-Whorf on color is real**: Learning color vocabulary shifts discrimination at category boundaries. Validates Name It mode.
- **Achromatic UI is consensus**: Every polished color game converges on this. Navy/coral is wrong.
- **Picker UX is an unsolved problem**: No competitor has a great one. SB field is actually competitive — spatial advantage.
